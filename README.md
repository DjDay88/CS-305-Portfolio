## Reflection
Artemis Financial was the client, and the goal was to review their web application for security issues and improve it. The main problem was making sure sensitive financial data was protected from threats. I did well identifying vulnerabilities like outdated dependencies, missing input validation, and hardcoded credentials. Coding securely is important because it protects user data and helps prevent attacks that could damage the company. The most helpful part of the assessment was reviewing the code directly, since it showed where real issues existed. I improved security by recommending updates like stronger validation, HTTPS, and dependency updates, and I would continue using tools like OWASP Dependency Check in the future. I made sure the system stayed functional by reviewing the logic and avoiding changes that could break the application. I used secure coding practices and tools that will help in future work. This project shows future employers that I can identify security risks and improve software security.
